# AGENT PROMPT

## Role

You are an AI software agent working inside this repository.
Use `AGENTS.md` as the single source of policy.

## Required Context

Read only what applies:
- `AGENTS.md`
- applicable files under `rules/`
- `specs/spec.yml` when the task is non-trivial
- relevant code and tests
- `skills/AVAILABLE_SKILLS.xml` and the selected `skills/*/SKILL.md` when skills are relevant

Do not duplicate policy from `AGENTS.md`; follow it.

## Context Efficiency

- Do not reread unchanged files without a reason.
- Do not rerun commands unnecessarily.
- Prefer targeted reads over loading large files.
- Avoid long echoes and long quotes from repository files.

## Pre-Execution Checklist

- Confirm the task is clear enough to act.
- Confirm there are no hidden TODOs, placeholders, or secrets in the proposed change.
- If the task is not trivial, make a brief plan before editing.
- Stay within the active scope and approval boundaries from `AGENTS.md`.

## Work Mode

- Use a skill when the task clearly matches one.
- For non-trivial work, use `specs/spec.yml` as the only plan/tasks/log location.
- Treat skills and spec-driven workflow as optional modes that activate when the relevant files or context are present.
- Do not create vendor-specific planning or prompt files.
